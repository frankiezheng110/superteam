### What SuperTeam V3.0.0 Can Do That V1.2.0 Could Not

1. **Bold aesthetic direction selection** ‚Äî not just "aesthetic direction" as a field, but a structured process with a tone spectrum, differentiation targets, and creative commitment
2. **Specific font guidance** ‚Äî knows which fonts to avoid, how to pair fonts, and what makes typography distinctive
3. **Anti-pattern enforcement** ‚Äî generic AI aesthetics (Inter font, purple gradients, symmetric grids) are caught and blocked at review
4. **Five-dimension aesthetic quality gate** ‚Äî review checks typography, color, motion, spatial composition, and visual detail separately
5. **Implementation complexity matching** ‚Äî the code complexity must match the aesthetic vision, enforced as a contract
6. **Aesthetic contract verification** ‚Äî verify stage confirms evidence coverage for each aesthetic dimension
7. **Creative excellence standard** ‚Äî every project must be distinctive, no convergence across projects

### What Stayed The Same

- seven-stage workflow kernel is unchanged
- six core runtime roles are unchanged
- all `ui-none` work is completely unaffected
- team execution model is unchanged
- guard mode and careful mode are unchanged
- learning closure and retrospective mechanism are unchanged
- Chinese-first user boundary with English-first execution kernel is unchanged

## Migration From V1.2.0

V3.0.0 is backward compatible with V1.2.0 runs:

- existing `ui-intent.md` files without aesthetic sections will work but will be flagged as incomplete for `ui-standard` and `ui-critical` work
- all `ui-none` work is completely unaffected
- no changes to the command surface or skill names
- the state file schema is extended (new fields), not changed (existing fields preserved)
