# SuperTeam V3.0.0 Release Notes

## Release Date

2026-03-23

## Summary

V3.0.0 完成了 `frontend-design` 前端美学能力与 SuperTeam 七阶段流程的深度融合。这不是简单的复制粘贴——`frontend-design` 的美学智能被完全解构后，分别融入了七个阶段的每一层，形成了一个完整的前端美学管线。

## Core Change: Frontend Aesthetics Deep Fusion

### What Was Absorbed From frontend-design

| 原始能力 | 融合位置 |
| --- | --- |
| Design Thinking (Purpose, Tone, Constraints, Differentiation) | `clarify` stage, `design` stage, `design-consultation` skill |
| Aesthetic Direction Selection (tone spectrum) | `design-consultation` skill, `designer` agent, `framework/frontend-aesthetics.md` |
| Typography Guidelines | all 7 stages via `ui-intent.md` contracts, `framework/frontend-aesthetics.md` |
| Color & Theme Guidelines | all 7 stages via `ui-intent.md` contracts, `framework/frontend-aesthetics.md` |
| Motion Guidelines | `plan`, `execute`, `review`, `verify` via contracts, `framework/frontend-aesthetics.md` |
| Spatial Composition | `design`, `execute`, `review` via contracts, `framework/frontend-aesthetics.md` |
| Backgrounds & Visual Details | `execute`, `review` via contracts, `framework/frontend-aesthetics.md` |
| NEVER List (anti-patterns) | anti-pattern registry in `framework/frontend-aesthetics.md`, enforced in `execute`, `review`, `verify` |
| Implementation Complexity Matching | `plan`, `execute` via contracts, `framework/frontend-aesthetics.md` |
| Creative Excellence Standard | `designer` agent, `design-consultation` skill |

### What SuperTeam V3.0.0 Can Do That V1.2.0 Could Not

1. **Bold aesthetic direction selection** — not just "aesthetic direction" as a field, but a structured process with a tone spectrum, differentiation targets, and creative commitment
2. **Specific font guidance** — knows which fonts to avoid, how to pair fonts, and what makes typography distinctive
3. **Anti-pattern enforcement** — generic AI aesthetics (Inter font, purple gradients, symmetric grids) are caught and blocked at review
4. **Five-dimension aesthetic quality gate** — review checks typography, color, motion, spatial composition, and visual detail separately
5. **Implementation complexity matching** — the code complexity must match the aesthetic vision, enforced as a contract
6. **Aesthetic contract verification** — verify stage confirms evidence coverage for each aesthetic dimension
7. **Creative excellence standard** — every project must be distinctive, no convergence across projects

### What Stayed The Same

- seven-stage workflow kernel is unchanged
- six core runtime roles are unchanged
- all `ui-none` work is completely unaffected
- team execution model is unchanged
- guard mode and careful mode are unchanged
- learning closure and retrospective mechanism are unchanged
- Chinese-first user boundary with English-first execution kernel is unchanged

## Anti-Pattern Registry

| ID | Anti-Pattern | Severity |
| --- | --- | --- |
| AP-01 | Generic fonts (Inter, Roboto, Arial, system) as primary | blocking |
| AP-02 | Purple gradients on white backgrounds | blocking |
| AP-03 | Predictable symmetric card grids | blocking |
| AP-04 | Cookie-cutter components without context character | blocking |
| AP-05 | Converging on same aesthetic across projects | blocking |
| AP-06 | Pure #000/#FFF without tonal softening | advisory |
| AP-07 | Identical animation timing on all elements | advisory |
| AP-08 | Decoration contradicting chosen aesthetic | advisory |
| AP-09 | Missing hover/focus states on interactive elements | advisory |
| AP-10 | No visual hierarchy in text sizing | advisory |

## Migration From V1.2.0

V3.0.0 is backward compatible with V1.2.0 runs:

- existing `ui-intent.md` files without aesthetic sections will work but will be flagged as incomplete for `ui-standard` and `ui-critical` work
- all `ui-none` work is completely unaffected
- no changes to the command surface or skill names
- the state file schema is extended (new fields), not changed (existing fields preserved)
