# SuperTeam V4.1.1 Release Notes

Date: 2026-03-26

## What Changed

- restored a stronger TDD contract for code-changing work
- made `red -> green -> refactor` explicit across planning, execution, review, and verification rules
- upgraded missing or violated failing-test-first behavior from a soft preference to a formal workflow issue
- kept the polish layer and seven-stage workflow unchanged

## Why This Version Exists

V4.1.1 restores the stronger SuperPowers-style TDD discipline without adding new top-level stages or extra standalone TDD agents.

The goal is to keep test-first behavior explicit, reviewable, and enforceable across the existing SuperTeam chain.

## Deliberate Non-Changes

- no new stage was added
- no standalone `tdd-guardian` agent was introduced in this release
- `inspector` still owns `review` and `verifier` still owns final verdicts
