# SuperTeam V4.2.0 Release Notes

Date: 2026-03-26

## What Changed

- rewrote Stage-2 external search around requirement anchors instead of project-name or source-first assumptions
- made keyword web search (`Google` or equivalent) plus GitHub keyword search the default first tier for brand-new product discovery
- moved official docs into the dependency-constraint validation layer instead of pretending a new internal product already has official materials
- added a dedicated `framework/solution-search.md` contract and upgraded `solution-landscape.md` / evidence-card expectations

## Why This Version Exists

V4.2.0 fixes the mismatch between source-first search rules and the reality of designing brand-new products.

The new rule searches for mature patterns around clarified needs, workflows, roles, constraints, and decision points, which makes Stage-2 search operational instead of ceremonial.

## Deliberate Non-Changes

- no new stage was added
- `design` still owns Stage-2 solution work
- `researcher` still supports design instead of replacing design judgment
