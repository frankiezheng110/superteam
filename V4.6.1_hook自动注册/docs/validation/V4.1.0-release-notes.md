# SuperTeam V4.1.0 Release Notes

Date: 2026-03-26

## What Changed

- added three new polish-layer agents: `simplifier`, `doc-polisher`, `release-curator`
- integrated a post-execute refinement bridge recorded in `.superteam/runs/<task-slug>/polish.md`
- kept the canonical seven-stage workflow unchanged while refining the `execute -> review` bridge
- taught `reviewer` to consume `polish.md` as an input without surrendering review ownership

## Why This Version Exists

V4.1.0 makes SuperTeam delivery outputs cleaner before formal review without weakening the reviewer or verifier authority model.

The result is a more publishable delivery surface and a more disciplined separation between refinement and strict review.

## Deliberate Non-Changes

- no new top-level stages were added
- `reviewer` still owns `review`
- `verifier` still owns final `PASS` / `FAIL` / `INCOMPLETE`
